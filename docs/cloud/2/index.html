在华为云部署高可用的DNS服务

需要的技术组件

  • 华为云弹性服务器

    推荐使用广州-public里的“CentOS-7.6-Docker-JDK-LVM-SWAP”私有镜像 申请2台服务器创建：私有镜像基于CentOS 7.6（x86_64-Minimal-1810）的ISO光盘部署，默认配置了SAWP分区，按LVM分区（Logical_Volume_Manager方便日后服务运行时可以系统不停机对分区扩容），安装了JDK 1.8 ，Docker 18.0，Docker-compose 1.14.0 ，软件源使用华为开源镜像站，参照《基于VirtualBox使用ISO创建Linux镜像》创建。

  • 华为云对象存储

    在华为云对象存储控制台中建立 obs-app-backup桶，用来存储2台DNS服务器的共有配置文件（dnsmasq.conf）。

  • 域名解析服务

    使用dnsmasq对应的docker版本jpillora/dnsmasq。提供简单的web界面可进行DNS的配置管理、日志查看等操作。

  • 反向代理服务

    Nginx 版本1.16.0 ，反向代理功能需要支持stream模块。

  • 高可用软件

    Keepalived是基于vrrp协议的一款高可用软件，为两台服务器提供一个共同的虚拟ip---VIP，平时VIP在主服务器上，当主服务宕机，VIP就跑到备用服务器上，引导客户端访问流量到备用服务器，从而实现高可用。Keepalived的HA分为抢占模式和非抢占模式，抢占模式即MASTER从故障中恢复后，会将VIP从BACKUP节点中抢占过来。非抢占模式即MASTER恢复后不抢占BACKUP升级为MASTER后的VIP。 本例中采用非抢占模式，这样会避免VIP切换可能造成的服务延迟。

挂载obs-app-backup桶(在2台服务器上)

参考《新宝骏App的对象存储使用方法》，挂载obs-app-backup桶到/dns ，可以使用本项目中的mount-obsfs.sh文件（需要自行更新AK/SK）。

部署Dnsmasq(在2台服务器上)

创建简单的配置 /dns/dnsmasq/dnsmasq.conf ：

#输出查询日志信息
log-queries
#do not use hosts nameservers
#默认会使用网关server,若需要配置本地局域网自动使用使用本DNS服务，则须将此选项设置为NO
no-resolv 
#配置使用的服务，即本地查询不到时，可通过此服务依次进行查询解析，可配置多个，一般为已知的或代理的外网DNS服务
server=100.125.1.250
server=100.125.136.29
server=223.5.5.5
#server=/syq/192.168.248.137
#explicitly define host-ip mappings
address=/dns.sgmw.com.cn/10.14.2.222

运行容器：

docker run \
    --name dnsmasq \
    -d \
    -p 53:53/udp \
    -p 8080:8080 \
    -v /dns/dnsmasq/dnsmasq.conf:/etc/dnsmasq.conf \
    -v /etc/localtime:/etc/localtime:ro \
    -e "HTTP_USER=***" \
    -e "HTTP_PASS=***" \
    --restart always \
    jpillora/dnsmasq

访问web UI

访问http://dns-host:8080，使用帐号/密码登录，进入类似如下界面，可直接修改配置，点击save即可快速生效。

部署Nginx(在2台服务器上)

安装Nginx

wget  http://nginx.org/packages/centos/7/noarch/RPMS/nginx-release-centos-7-0.el7.ngx.noarch.rpm
  rpm -ivh nginx-release-centos-7-0.el7.ngx.noarch.rpm
  yum install nginx -y
  systemctl start nginx
  systemctl enable nginx

配置Nginx

cd /etc/nginx/conf.d/
  cp nginx.conf nginx.conf.bak
  nano nginx.conf

反向代理配置，复制到nginx.conf：

#user nobody;
worker_processes 4;
events {
worker_connections 1024;
}
stream {
upstream dns {
server 10.14.2.21:5353 weight=1;
server 10.14.2.63:5353 weight=1;
}
server {
listen 53 udp;
proxy_connect_timeout 1s;
proxy_timeout 20s;
proxy_pass dns;
}
}

启用配置

nginx -t
  nginx -s reload

Keepalived(在2台服务器上)

系统环境准备

systemctl stop firewalld.service
systemctl disable firewalld.service
sed -i 's#SELINUX=enforcing#SELINUX=disabled#g' /etc/selinux/config
setenforce 0

安装Keepalived

yum install keepalived -y
  mkdir /etc/keepalived
  nano /etc/keepalived/keepalived.conf

主节点配置

! Configuration File for keepalived
global_defs {
    router_id node_01
}
vrrp_script chk_nginx {
    script "killall -0 nginx && netstat -an | grep :::5353"
    interval 1
    weight - 5
    fall    3
    rise    5
}
vrrp_instance VI_1 {
    state MASTER
    interface eth0
    virtual_router_id 88 
        priority 100 
        advert_int 1 
    nopreempt
    authentication {
        auth_type PASS
        auth_pass 8888
    }
    virtual_ipaddress {
        10.14.2.222
    }
    track_script {
        chk_nginx
    }
}

备节点配置

! Configuration File for keepalived
global_defs {
    router_id node_02
}
vrrp_script chk_nginx {
    script "killall -0 nginx && netstat -an | grep :::5353"
    interval 1
    weight - 5
    fall    3
    rise    5
}
vrrp_instance VI_1 {
    state BACKUP
    interface eth0
    virtual_router_id 88 
        priority 99 
        advert_int 1 
    nopreempt
    authentication {
        auth_type PASS
        auth_pass 8888
    }
    virtual_ipaddress {
        10.14.2.222
    }
    track_script {
        chk_nginx
    }
}

启用配置

systemctl start keepalived 
  systemctl enable keepalived
  systemctl status keepalived -l
